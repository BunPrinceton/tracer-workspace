I. Quick Start: Drag & Drop Method

The Easiest Way - 5 Simple Steps!

Total Time: ~2 minutes

Difficulty: Beginner-friendly

Best For: First-time users who want to get started immediately

Step-by-Step Quick Installation

1 Install Tampermonkey Extension

Choose your browser:

  • Chrome: Chrome Web Store
  • Firefox: Firefox Add-ons
  • Edge: Edge Add-ons
  • Safari: Mac App Store
  • Opera: Opera Add-ons

Click "Add to [Browser]" and confirm the installation.

2 Download Your Userscript

Save the .user.js file to your computer (usually in your Downloads folder).

Important: The file MUST end with .user.js (not just .js) for drag-and-drop to work!

3 Open Tampermonkey Dashboard

Method 1: Click the Tampermonkey icon in your browser toolbar → "Dashboard"

Method 2: Navigate to chrome://extensions (or your browser's equivalent) → Find Tampermonkey → Click "Details" → Click "Extension options"

4 Drag and Drop the Script File

How to Drag & Drop:

  1. Open your file explorer (where you downloaded the .user.js file)
  2. Position the file explorer window next to your browser
  3. Drag the .user.js file from your file explorer
  4. Drop it anywhere on the Tampermonkey Dashboard page
  5. A new tab will open showing the script installation screen

5 Install the Script

On the installation screen, you'll see:

  • The script name and description
  • What websites it will run on (@match URLs)
  • What permissions it needs (@grant)

Click the big "Install" button to add the script to Tampermonkey!

Post-Installation Quick Checks

Verify Installation (30 seconds):

  • Refresh the target webpage (Ctrl+Shift+R or Cmd+Shift+R)
  • Click the Tampermonkey icon - you should see your script listed with a checkmark
  • Check the script counter badge on the Tampermonkey icon (e.g., "1" means 1 script active)

Enable Developer Mode (If Script Not Working)

Chrome/Edge/Brave Browser Extension Developer Mode:

  1. Go to chrome://extensions (or edge://extensions or brave://extensions)
  2. Toggle "Developer mode" in the top-right corner to ON
  3. Refresh your webpage

*Firefox users: Firefox doesn't require Developer Mode for Tampermonkey - scripts work right out of the box!

Enable "Allow User Scripts" (Brave/Chrome)

Critical Step for Brave Browser Users: After enabling Developer Mode, you must also enable user scripts for Tampermonkey.

  1. While still on brave://extensions (or chrome://extensions)
  2. Find Tampermonkey in the list of extensions
  3. Click "Details" button on the Tampermonkey card
  4. Scroll down to find "Allow User Scripts" toggle
  5. Toggle it to ON
  6. You'll see a warning: "The extension will be able to run code which has not been reviewed by Brave. It may be unsafe and you should only enable this if you know what you are doing."
  7. This is expected and safe for Tampermonkey - this is literally what Tampermonkey is designed to do!
  8. Refresh any webpage where you want scripts to run

Note: Without this setting enabled, your userscripts will NOT run in Brave browser, even if everything else is configured correctly.

How to Verify Your Script Is Running

Method 1: Hover Over Tampermonkey Icon

When you're on a webpage where a script should run, hover your mouse over the Tampermonkey icon in your browser toolbar. A tooltip will show which scripts are currently active.

Method 2: Check the Console (F12)

Press F12 to open Developer Tools, then click the "Console" tab.

Run this command to verify scripts are loaded:

// Check if Tampermonkey is injecting scripts

console.log('Tampermonkey active scripts:', window.GM_info);

// If you see script details, Tampermonkey is working!

// If you see "undefined", scripts may not be injecting properly

Done! Your Script Is Installed!

You're all set! Your Tampermonkey userscript should now be running.

II. Helpful Console Commands

How to Use These Commands

  1. Press F12 to open Developer Tools
  2. Click the "Console" tab
  3. Copy and paste any command below
  4. Press Enter to run it

Check If Tampermonkey Is Active

// Check if Tampermonkey APIs are available

if (typeof GM_info !== 'undefined') {

    console.log('✅ Tampermonkey is active!');

    console.log('Script info:', GM_info);

} else {

    console.log('❌ Tampermonkey is NOT active on this page');

}

List All Active Scripts

// Display all running scripts

if (GM_info && GM_info.scripts) {

    console.log('📜 Active scripts on this page:');

    GM_info.scripts.forEach(script => {

        console.log(`- ${script.name} (v${script.version})`);

    });

} else {

    console.log('No Tampermonkey scripts detected');

}

Check Current Page URL

// Verify what URL Tampermonkey sees

console.log('Current URL:', window.location.href);

console.log('Protocol:', window.location.protocol);

console.log('Hostname:', window.location.hostname);

console.log('Port:', window.location.port);

// This helps debug @match URL patterns

Test If Script Should Match Current Page

// Example: Check if localhost URL should match

const currentURL = window.location.href;

console.log('Current URL:', currentURL);

// Check against common @match patterns

const patterns = [

    'http://localhost:9090/*',

    'http://127.0.0.1:9090/*',

    'https://webknossos.org/*'

];

patterns.forEach(pattern => {

    const regex = new RegExp(pattern.replace(/\*/g, '.*'));

    if (regex.test(currentURL)) {

        console.log(`✅ URL matches: ${pattern}`);

    } else {

        console.log(`❌ URL does NOT match: ${pattern}`);

    }

});

Force Reload Scripts

// Hard refresh the page to reload all scripts

location.reload(true);

// Alternative: Clear cache and reload

window.location.href = window.location.href;

Check for Script Conflicts

// List all global variables (can reveal conflicts)

console.log('Global variables:', Object.keys(window));

// Check if specific script functions exist

if (typeof someScriptFunction !== 'undefined') {

    console.log('✅ Script function is loaded');

} else {

    console.log('❌ Script function is missing');

}

Pro Tip: Add Debug Logging to Your Script

Add this line at the top of your userscript to confirm it's loading:

console.log('✅ [SCRIPT NAME] loaded successfully!');

If you see this message in the console, you know the script is at least being injected, even if it's not working as expected.
